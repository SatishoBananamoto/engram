# Introduced project scoping with .svx/ directory initialization

## Context

SVX lacked context awareness and was blocking operations outside of relevant projects. This was identified as "Problem 1 (no context awareness)" in the rework plan.

## Choice

Implemented project scoping system:
- `svx init` creates .svx/ directory and sets mode
- Updates .gitignore automatically
- Hook auto-allows operations outside .svx/ projects

## Reasoning

SVX should only guard opted-in projects rather than blocking everything globally. This provides better user control and reduces false positives for operations in unrelated directories.

## Alternatives Considered

No alternatives mentioned in the commit message, though this addresses "Problem 1" from a broader rework analysis.
