# Built caliber from MY UNIVERSE calibration data, not from spec

## Context

PLAN-TRUST-LAYER.md had a detailed spec for caliber — API draft, file list, phased roadmap. Could have built from that spec top-down. Instead built by extracting patterns from real CALIBRATE.md data (36 predictions).

## Choice

Build from data, not from spec. First artifact was the extraction script, not the API. The Trust Card format emerged from what the data actually needed to show.

## Reasoning

Spec-first would have baked in session 1's incorrect finding ("70-79% is the danger zone"). Data-first revealed the correction: 60-69% is the actual danger zone. The spec assumed the analysis was right. The data proved it wasn't.

This validates the broader caliber thesis: you need enough data AND the right aggregation format to see real patterns. Point-in-time analysis is unreliable.
