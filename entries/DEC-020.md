# Changed from binary block/allow to advisory denies with risk classification

## Context

SVX was blocking legitimate operations in practice. User reported that "SVX blocked legitimate operations" during a parallel session building scroll. The system used binary decisions without context awareness and stateless checks.

## Choice

Implemented advisory vs hard deny system with risk classification:
- Added DenyKind enum and advisory_action field
- Created _build_advisory_action function for concrete instructions on git, file, config operations
- Added _classify_target function for file risk scoring by type and location
- Added user-friendly verdict formatting

## Reasoning

Binary decisions were too restrictive and lacked nuance. Risk classification allows the system to provide helpful guidance while still protecting against catastrophic operations. Advisory denies give users actionable information rather than just blocking them.
