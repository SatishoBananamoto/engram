# Calibration bucket patterns are not statistically significant — need 100+ predictions per bucket

## Context

Ran binomial tests on all calibration bucket findings from 59 verified predictions. None are significant at p<0.05 or even p<0.10. The evidence quality model (3/3 vs 0/3) has p=0.100 — suggestive only.

## What Broke

Built detailed cognitive models on data that can't distinguish from noise. Three sessions of analysis might be pattern-finding in random data. The "danger zone" concept, the evidence quality model, and the domain improvement trends — all underpowered.

## How to Prevent

Check statistical significance BEFORE building models. With 16 predictions in a bucket, you need a very large effect to detect significance. For moderate effects (8-point calibration gap), need ~100+ predictions per bucket. caliber should automatically run significance tests on Trust Card data and flag underpowered buckets.
