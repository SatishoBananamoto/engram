# The predecessor had 70 modules analyzing 5K lines of content

## Context

Reviewed Claude-owns-this repo before building engram.

## Observation

31,135 lines of Python code analyzing 4,641 lines of markdown content. The tools-to-content ratio was 6.7:1. The system spent more energy analyzing itself than knowing things.

## Significance

This is the canonical failure mode of self-referential systems: the meta-layer outgrows the object-layer. Engram must avoid this. The current ratio after building parser, validator, graph, query, health, and CLI is roughly 1:1 (tools ~600 LOC, entries ~100 lines). That's healthier, but needs watching.
