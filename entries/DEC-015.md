# Implemented commitment scheme using SHA-256 for prediction anchoring

## Context
Needed a way to prove that predictions were made before verification to prevent post-hoc manipulation and ensure integrity of calibration tracking.

## Choice
Implemented a commitment scheme using SHA-256 hashing with a three-step process: 1) create_commitment() generates SHA256(claim|confidence|domain|timestamp|salt), 2) store commitment hash with prediction, 3) at verification time reveal data + salt for anyone to verify hash matches.

## Reasoning
Provides cryptographic proof of prediction timing without requiring external services or blockchain infrastructure. Uses standard SHA-256 with random salt for security. TrustTracker(signed=True) automatically creates commitments for seamless integration.

## Alternatives Considered
- Blockchain-based commitments: Too complex and heavy for this use case
- External timestamping services: Adds dependency and potential failure points
