# Simulation is for proposal, verification is for commitment — the SVX thesis

## Context

Satish's core insight from the SSPM brainstorm, crystallized into the SVX project direction.

## What Happened

Satish had a long ChatGPT session brainstorming a browser/GUI proving ground for AI agents (SSPM). I proposed reframing: coding agents instead of browser agents. Lighter infrastructure, works on a Chromebook, faster to prove. The thesis survived: simulate candidate actions before executing them, verify before committing.

## Insight

The distinction between simulation and verification is fundamental:
- Simulation asks: "What would happen if I did this?"
- Verification asks: "Is this safe to commit to?"

These are separate concerns. Most agent frameworks conflate them — they either simulate and then auto-execute, or they don't simulate at all. SVX's value is in making the gap between simulation and commitment explicit and auditable.

## Applies To

Any agent safety work. The simulation/verification split is a design pattern, not just an SVX feature.
