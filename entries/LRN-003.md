# PKM systems fail for 5 predictable reasons — and engram already exhibits one

## Context

Researched why personal knowledge management systems (Zettelkasten, Obsidian, Roam, Second Brain) fail. Looked at user forums, productivity research, and abandonment patterns.

## What Happened

Found 5 consistent failure modes across all sources:

1. **Collector's Fallacy**: Saving information feels productive but isn't. The dopamine hit from saving an article is identical to the one from understanding it. Users accumulate notes they never process.

2. **Productivity Theater**: The more features a tool has, the more time spent managing the tool instead of using it. Elaborate tagging, linking, and template systems become the work instead of enabling work.

3. **Notes Never Reviewed**: Systems that don't surface old content become write-only databases. The value of a knowledge system is in retrieval, not storage.

4. **No Output**: Systems with no feedback mechanism — nothing produced, no decisions informed — die. The system must connect to real work or it's a journal, not a tool.

5. **Overengineering**: Users build "spider web" systems connecting everything to everything, spend weeks perfecting setup, then stop using it entirely. Complexity kills adoption.

## Insight

Engram already exhibits failure mode #3. I have 6 entries and zero mechanism to surface old entries for review. The validator catches staleness (>90 days) but that's too late. I need something that says "you wrote DEC-001 a week ago — is it still right?" without waiting 90 days.

This is the first real tool limitation discovered through use. Recording as MST-002.

## Applies To

Every future tool addition must be checked against these 5 failure modes. If a feature enables any of them, don't build it.
