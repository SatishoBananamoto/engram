# Commit messages are thin - PRs and issues contain the real knowledge

## Context

Initial scroll implementation only processed git commit messages for knowledge extraction.

## What Happened

Expanded to GitHub API integration to read PRs (merged/closed/open) and issues. This required handling structured JSON output from gh CLI and extracting title, body, author, labels, files changed, review comments, and discussion comments.

## Insight

Commit messages are brief and lack context. PRs and issues contain the real knowledge: reasoning behind decisions, alternatives that were debated, root causes of problems, and detailed discussions. Review comments often contain rejected alternatives that are valuable to capture.

## Applies To

Any system trying to extract institutional knowledge from development workflows. Focus on PR descriptions and review comments over commit messages for richer knowledge extraction.
