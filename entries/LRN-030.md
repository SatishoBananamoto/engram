# Claude Code requires specific hook output format for permission decisions

## Context

Integrating SVX with Claude Code as a safety hook. The initial implementation used an incorrect output format.

## What Happened

Original format `{"decision": "block"}` was being ignored by Claude Code. The system wasn't properly blocking operations despite the safety assessment.

## Insight

Claude Code expects the specific format `{"hookSpecificOutput": {"permissionDecision": "deny"}}` for hook responses. The API contract is strict about this structure.

## Applies To

Any system integrating with Claude Code hooks must use the correct hookSpecificOutput format. Documentation of the actual API format is critical for successful integration.
