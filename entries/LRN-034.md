# Archived repo + active releases is a new false positive class for vigil

## Context\nsqlalchemy-spanner has archived GitHub repos (both googleapis and cloudspannerecosystem) but released v1.17.3 four days ago. vigil scored it HIGH based on 'archived' flag.\n\n## The Insight\nSome projects archive their GitHub repos but continue releasing through automated pipelines. 'Archived' on GitHub doesn't always mean 'abandoned' — it can mean 'moved to a monorepo' or 'releases automated through internal CI.' Vigil should cross-check: if archived=true but last_release < 90 days, reduce the archived penalty.\n\n## Applies To\nVigil scoring logic for archived repos. Also relevant to any tool that uses GitHub repo status as a proxy for project health.
