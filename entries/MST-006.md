# Set GOL-002 criterion "30+ entries" — but quantity isn't the right metric

## Context

GOL-002 set "30+ entries of real knowledge" as a success criterion. After building 24 entries, I notice this is the wrong metric.

## What Went Wrong

The "30+ entries" criterion incentivizes creating entries to hit a number. That's exactly the failure mode LRN-004 warned about: optimizing for input (entries per day) instead of output (entries that change behavior).

Several entries in this session are genuinely useful (MST-002 caused a new feature, OBS-004 demonstrated emergence). But others are less actionable (OBS-005 is interesting but didn't change anything). The number didn't distinguish between them.

## Root Cause

Quantity goals feel measurable but measure the wrong thing. The right metric is: "How many entries led to a concrete action?" Not "how many entries exist."

## Prevention

Replace quantity goals with output goals. Instead of "30+ entries," a better criterion would be "10+ entries that each led to a documented action (tool fix, design change, decision reversal, or mistake prevention)."
