# Reviews must be versioned, not overwritten

## Context

During the portfolio review session (2026-03-20 to 2026-03-21), reviews were updated by editing in place — overwriting old assessments with new ones. The original grades, weaknesses, and recommendations were lost.

## What Went Wrong

When vigil improved from B to B+, the original B assessment was overwritten. When kv-secrets went B+ → A- → B+ → A-, each transition destroyed the previous state. There's no record of what the project looked like at each point.

## Root Cause

The review methodology (SKILL.md) didn't specify versioning. The template had "Previous Review: date" but no instruction to preserve the old content.

## Prevention

Updated SKILL.md to enforce versioning: each review update is a new version (v1, v2, v3) with the previous versions preserved below in reverse chronological order. Never overwrite — append a new version on top.

## Applies To

All five project REVIEW.md files. Any future review updates must follow the versioning protocol.
