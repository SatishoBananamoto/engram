# Published as mcp-probe on PyPI instead of probe due to name conflict

## Context

The project was initially called 'probe' but during PyPI release preparation, a naming conflict was discovered.

## Choice

## Reasoning

The name 'probe' was already taken on PyPI, requiring an alternative name for the package.

## Alternatives Considered

No explicit alternatives mentioned in the commit history, but 'mcp-probe' was chosen to maintain the MCP (Model Context Protocol) association while resolving the naming conflict.
