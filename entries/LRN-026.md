# HTTP redirects can cause analyzer crashes without proper handling

## Context

During development, the team encountered crashes when analyzing certain packages that had repository URL redirects.

## What Happened

The deep-translator package caused a 301 redirect crash because the HTTP client wasn't configured to follow redirects automatically.

## Insight

Package repository URLs can change over time, and services like GitHub frequently use redirects. Analyzers must handle HTTP redirects gracefully to maintain robustness when processing real-world package metadata.

## Applies To

Any system that fetches data from package repositories or follows URLs from package metadata. Proper HTTP redirect handling is essential for reliability.
