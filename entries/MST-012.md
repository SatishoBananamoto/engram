# Importer regex was too strict, causing data import failures

## Context
The prediction data importer was successfully processing only 59 out of 87 available entries, indicating a parsing problem.

## What Went Wrong
The importer regex required double newlines after headers, but some entries in the data had only single newlines after headers.

## Root Cause
Overly strict regex pattern that didn't account for variation in entry formatting.

## Prevention
Made the regex more flexible to handle entries with single newline after header, improving import success rate to 77/87 entries.
