# Absence of data is a signal, but severity must be conditional

When vigil can't find a GitHub repo for a package, that's a sustainability signal — but the severity depends on context. Old packages with no repo link AND stale releases are likely abandoned (hard penalty). Packages with no repo link but recent releases are just using old-style PyPI metadata (soft penalty).

A flat penalty for missing data caused false alarms on numpy, rich, and colorama. Making the penalty conditional on release recency eliminated false positives while still catching genuinely abandoned packages (pycrypto, nose).

Broader principle: missing data signals should be calibrated against what other data IS available, not applied uniformly.
