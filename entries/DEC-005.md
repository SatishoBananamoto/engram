# Engram should span projects, not live inside one

## Context

Engram currently lives in its own repo tracking its own build. But the real value is tracking decisions across svx, persona-engine, and whatever comes next.

## Choice

Engram's entries can reference any project via the `project:` field. Future entries about svx decisions, persona-engine learnings, etc. should live here. One knowledge base, many projects.

## Reasoning

Knowledge doesn't respect repo boundaries. A mistake made in persona-engine (MST-005: cascade collapse) applies to svx too. A design principle learned from engram (DEC-003: "so what?" test) applies to all projects. Splitting knowledge by repo fragments it.

## Alternatives Considered

1. Keep engram isolated — rejected: loses cross-project learning
2. Copy entries into each project repo — rejected: creates sync problems
3. Use engram as the central knowledge store, reference projects — chosen: simple, no duplication
