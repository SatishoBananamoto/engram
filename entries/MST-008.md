# Engram entries were bloated with implementation details

Logged DEC-007 and DEC-008 with class names, method signatures, CLI flags, confidence values, file paths — all of which the code already says. Engram should capture WHY a decision was made and what tradeoff was accepted, not WHAT was built.

Two tests before writing an engram entry:
1. "So what?" — if removing this entry wouldn't cause someone to make a wrong decision, it doesn't belong
2. "One-line test" — if the code already says it, engram shouldn't

The tradeoff reasoning in DEC-008 (why PyPI-only for transitive deps) was the only valuable part. The rest was noise.
