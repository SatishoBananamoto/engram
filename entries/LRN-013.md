# Vigil session protocol — read before build, update before commit

## What Happened

In Session 2, I built the entire rate limiting feature (config module, auth CLI, budget tracking, 20 tests) before updating Craft.md. Satish caught this — same mistake as Session 1. The process docs exist to enable session handoff across devices and Claude instances. If they're stale, the next session starts blind.

## Rule

Every vigil session MUST follow this protocol:

### Session Start
1. Read `Craft.md` — current phase, pending items, decision log
2. Read `REVIEW.md` — priorities, weaknesses, recommendations
3. Read `Skills.md` — gotchas, API patterns, things learned the hard way
4. Check engram for relevant cross-project learnings
5. Update Craft.md session log with "Session N started, objective: X"

### During Work
- Update Craft.md phase checklist as items are completed
- Log decisions to Craft.md decision log AND engram
- Log learnings/mistakes to Skills.md AND engram

### Before Commit
- Update Craft.md (phase status, session log, decision log)
- Update Skills.md if new patterns/gotchas discovered
- Update REVIEW.md if a weakness was addressed or new one found
- THEN commit

### Session End
- Write "Next:" line in Craft.md session log
- Ensure all three docs are current

## Why This Matters
Satish works across devices and sessions. These docs are the handoff protocol. A fresh Claude instance with stale docs will duplicate work, miss context, or repeat mistakes.
