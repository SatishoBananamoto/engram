# The real product is the agent-facing layer, not the extraction pipeline

## Context

Built scroll in two phases: extraction pipeline (Phase 1) and agent integration (Phase 2). The extraction pipeline was working and tested within the first hour. The agent integration — MCP server, export formats, relevance engine — is what transforms it from a CLI tool into something agents can actually use.

## What Happened

Phase 1 (extraction) validated that the idea works: LLM can extract structured knowledge from git commits. But the extracted knowledge sitting in markdown files is useless to agents unless there's a delivery mechanism. Phase 2 built three delivery paths:
1. CLAUDE.md export — zero-effort context injection
2. MCP server — agents query mid-session
3. Relevance engine — surfaces the right knowledge for the current task

## Insight

Extraction is table stakes. The competitive moat is in delivery: getting the right knowledge to the right agent at the right moment. A perfect extraction pipeline with no delivery mechanism is a research project. A decent extraction pipeline with good delivery is a product.

## Applies To

Any knowledge system. Engram learned this too (OBS-003: knowledge APPLICATION matters most). The pattern holds: capture is solved, delivery is the differentiator.
