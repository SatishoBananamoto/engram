# Built probe — MCP server security scanner

## Why probe exists

Research showed 43% of MCP servers have command injection vulnerabilities, 53% use plaintext API keys in config files, and 492 servers were found running without authentication. Nobody owns this problem — no scanner, no rating system, no audit tool.

probe fills the "trust the MCP servers your agent connects to" gap in the portfolio:
- svx: trust the agent won't break your code
- vigil: trust the dependencies the agent chooses
- kv-secrets: trust the agent with your secrets
- engram: trust your decision history
- scroll: trust the institutional context
- **probe: trust the MCP servers the agent connects to**

## What it does (v0.1.0)

5 scanners: secrets (prefix matching + Shannon entropy), injection (subprocess/eval/exec patterns), filesystem (path traversal, sensitive paths), validation (missing input checks in tool handlers), transport (HTTP vs HTTPS, sudo, unpinned packages).

A-F grading with weighted scoring. Rich terminal output + JSON mode. CI-friendly exit codes.

## Why NOT an LLM-based scanner

Deterministic pattern matching, same philosophy as svx. No API calls, no cost, runs in &lt;1s. LLM-based analysis could be a future layer but the base must be fast and free.
