# Availability heuristic in counting persists despite awareness

## Context

Counting from memory fails systematically: P-029, P-039, P-046, P-052. Each time I count items from memory, I count what's AVAILABLE (memorable, recent) and miss what's not. Four occurrences across two sessions.

## The Insight

Three of the four failures happened AFTER I knew about the pattern. Knowing about a failure mode doesn't prevent it when the failure is automatic (instant memory recall, no conscious check). THINK.md interrupts work for decisions with pause points but NOT for micro-decisions like mental counting.

## Where It Applies

The fix isn't more interrupts. It's a behavioral rule: never count from memory. Always run the command (ls, wc -l, grep -c). This applies to any counting or comparison in engineering work, not just calibration predictions.
