# Build a knowledge system that proves itself by being used

## Objective

Build a structured knowledge system (engram) that tracks decisions, learnings, mistakes, and observations — and validate it by using it during its own construction.

## Success Criteria

1. Parser correctly extracts all entries from markdown files with YAML frontmatter
2. Validator catches broken links, schema violations, duplicates, and stale entries — verified with intentionally bad data
3. Graph module identifies bridges, clusters, and connections — verified against known graph structures
4. Health score drops when the system is degraded and rises when it's healthy — verified by controlled degradation
5. The knowledge base contains real content from the build process, not manufactured filler
6. Every module is functionally tested: does it do what it claims, not just does the code run

## Current Status

- [x] Goal defined
- [x] Parser built and tested (30 tests — catches missing fields, bad types, empty files, prefix mismatches)
- [x] Validator built and tested (27 tests — catches broken links, duplicates, orphans, staleness, bad supersession)
- [x] Graph module built and tested (24 tests — verified against chain, triangle, star, disconnected structures)
- [x] Query/search built and tested (22 tests — search by id/type/tag/project, full-text, backlinks)
- [x] Health score built and tested (16 tests — score drops for every kind of degradation, verified)
- [x] CLI built and tested (7 commands, all working against real data)
- [x] System used to track its own build (6 real entries from the build process)
- [x] Loop closed: bridge detected → fixed → score improved (97 → 100)
