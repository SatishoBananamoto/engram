# Keep MY UNIVERSE and caliber separate, bridge with import

## Context

MY UNIVERSE uses CALIBRATE.md (narrative markdown) for calibration tracking. caliber uses JSON/FileStorage for data. Question: should caliber replace CALIBRATE.md as the backend?

## Choice

No. Keep separate, bridge with extract_calibrate_md.py import.

## Reasoning

CALIBRATE.md's narrative format (notes, batch analysis, hypothesis tests) IS the practice. caliber captures data but not reasoning. The notes between predictions are where thinking happens. Stripping them into a database would reduce the practice to data collection.

caliber handles machine analysis (Trust Cards, confidence buckets). MY UNIVERSE handles human-facing thinking (narrative, analysis). The bridge (import script) syncs data from narrative to machine-readable without losing the narrative.
