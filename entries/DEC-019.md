# Added MCP server mode to enable auditing other MCP servers

## Context

The probe tool was initially a CLI-only security scanner. There was a need to enable other MCP servers to audit MCP configurations programmatically.

## Choice

Implemented MCP server mode with 4 exposed tools: probe_scan, probe_scan_server, probe_list, and probe_check_config, along with a `probe serve` CLI command.

## Reasoning

MCP server mode allows other MCP servers to perform security audits programmatically, enabling integration into larger MCP ecosystems and automated workflows.

## Alternatives Considered

No explicit alternatives mentioned, but the decision maintains both CLI and server capabilities rather than replacing one with the other.
