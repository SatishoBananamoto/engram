# CrewAI has 3x the dependency risk of any other AI agent framework

## Context\nScanned 6 AI agent frameworks with vigil: CrewAI, LangChain Core, AutoGen Core, MCP SDK, A2A SDK, OpenAI Agents.\n\n## The Pattern\nCrewAI has 29 direct dependencies — 3-6x more than competitors. It's the only framework with CRITICAL (jsonref 0.78) and HIGH (tomli-w 0.68, openpyxl 0.64) risk packages. 127 transitive dependencies.\n\nLangChain Core has 8 deps, no HIGH or CRITICAL. A2A SDK and OpenAI Agents even leaner.\n\njsonref is a shared CRITICAL risk between CrewAI and AutoGen — abandoned with no maintainer activity.\n\nMCP SDK has 3 bus-factor-1 dependencies (typing-inspection, sse-starlette, jsonschema) — personnel risk, not code risk.\n\n## Why It Matters\nThis is vigil's first real-world finding on production software. The analysis is publishable and demonstrates vigil's value to the AI agent community. Dependency count correlates with risk — not just build complexity but security surface.
