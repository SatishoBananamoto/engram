# Bridge scroll to engram with deposit module for knowledge system interoperability

## Context

Scroll extracts knowledge from development history, while engram is an established knowledge management system. Need interoperability between the two systems.

## Choice

Built `scroll/deposit.py` that bridges scroll → engram. Reads scroll entries, assigns fresh engram IDs, writes engram-compatible markdown with `source: scroll` in frontmatter. Tracks deposits in state file to prevent duplicates.

## Reasoning

- Enables knowledge flow from automated extraction (scroll) to manual curation (engram)
- Preserves source provenance with scroll-extracted tag and source field
- Prevents duplicates through state tracking
- Maintains engram compatibility for existing workflows

## Alternatives Considered

- Replace engram entirely: would lose existing manual knowledge
- Manual copy/paste: error-prone and doesn't scale
- One-way sync only: limits knowledge management flexibility
