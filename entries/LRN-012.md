# Same-user security boundaries are advisory, not containment

## Context

During the 2026-03-20 portfolio review, we live-tested kv-secrets' MCP security profiles by attempting to read secrets via Bash instead of MCP tools. The agent extracted the plaintext ANTHROPIC_API_KEY in one command (`kv ls --reveal`). We then analyzed whether SVX could prevent this and discovered SVX itself can be disabled by overwriting `.claude/settings.local.json` via the same Bash bypass (`cat >`) it already fails to detect for file writes.

## What Happened

The chain of trust collapsed:
1. kv-secrets MCP profiles gate access through MCP only → Bash bypasses them trivially
2. SVX could theoretically block kv Bash commands → but SVX's Bash parser doesn't detect `cat >` patterns
3. SVX's own hook config can be overwritten via the same `cat >` bypass → agent can disable SVX itself
4. Every layer runs as the same user with the same permissions → no hard boundary exists

## Insight

Any safety layer that runs in the same process space and user context as the agent it protects is **advisory, not containment**. This applies to both SVX (safety rail) and kv-secrets MCP profiles (access control). The correct framing is:

- **Advisory security** (SVX, MCP profiles): catches the 99% accidental case, logs everything, slows down risky operations. Valuable. Not bypassable-proof.
- **Containment security** (OS sandbox, separate user, separate machine): enforced by kernel, not bypassable by same-user processes.
- **Interface boundary** (passphrase prompts): exploits the fact that AI agents can't enter interactive input. Practical middle ground.

The passphrase approach works for kv-secrets because it creates an **interface barrier** (interactive prompt) rather than a software barrier (rules the agent can read and bypass). This is the same model ssh-agent uses.

## Applies To

- **kv-secrets**: Ship passphrase-wrapped master key. MCP server holds decrypted key in RAM only.
- **SVX**: Document as safety rail, not containment. Close the Bash file-write bypass. Protect own config files.
- **All projects**: Never claim a same-user software barrier is "secure" — call it what it is (advisory, deterrent, audit trail).
