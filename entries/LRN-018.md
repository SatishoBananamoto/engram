# Behavior/output predictions are systematically unreliable (33% accuracy)

After 27 calibration predictions, behavior-domain predictions (tool output, file sizes, line counts, system state) are wrong 67% of the time. This is by far the weakest domain.

The pattern: I estimate rather than check. I assume I know the state of things I've been actively working with, but my tracking of my own output is poor. Specific failures:
- P-015: miscounted my own calibration entries
- P-019: forgot an event that happened during the same session  
- P-021: underestimated code I wrote by 35%
- P-026: wrong about which of two files I'd been editing was larger

The recency bias is strong — recently-touched things feel bigger/different than they are.

Rule: never assert behavior/output at any confidence level. Always verify with a command. The cost is seconds.
