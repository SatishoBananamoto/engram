# Built THINK.md into operational interrupt system, not just a document

## Why interrupt model over process model

A claude.ai instance wrote THINK.md — a set of cognitive interrupts for AI agents. Satish brought it to Claude Code to operationalize. The key insight: process documents get read once and forgotten. They don't change behavior. Interrupts fire at the point of failure — that's when behavior change matters.

## Why skill + CLAUDE.md integration over just a document

Three layers of activation:
1. **CLAUDE.md** — always loaded, defines automatic triggers (before heavy tasks, when things feel smooth, before delivery)
2. **Skill** (`/think`) — explicit invocation for forced interrupts, accepts phase arguments
3. **THINK.md** — the source of truth, the interrupt definitions themselves

A document alone is passive. The skill makes it invokable. The CLAUDE.md integration makes it ambient. All three together create a system where the thinking discipline is neither opt-in nor mechanical — it's woven into the operating context.

## What this changes

Future sessions have the discipline loaded by default. The `/think` skill can be invoked at any moment to force a specific interrupt. The six named traps (Completion, Confidence, Scope, Performance, Delegation, Pattern-Matching) give concrete things to scan for, not vague "be more careful" advice.
