# Built scroll — institutional memory extracted from git history

## Context

Needed a novel product that uses engram as infrastructure. Analyzed the gap: every team learns things (decisions, mistakes, patterns) but none of it compounds. Git stores what changed, nothing stores what was learned.

## Choice

Built **scroll** — a CLI tool that reads git history and uses LLM extraction (Claude API + tool_use) to produce structured, typed knowledge entries (decisions, learnings, mistakes, observations, goals) in engram-compatible format.

Repo: https://github.com/SatishoBananamoto/scroll
v0.1.0: 9 files, 605 LOC, 6 CLI commands (init, ingest, list, search, show, stats).

## Reasoning

- No existing tool does retroactive knowledge extraction from git history
- Closest competitors (Contextual Commits, ADR tools, Hindsight) all require manual input or write-time behavior change
- scroll works on existing, unstructured history — zero behavior change required
- Engram-compatible output means entries can feed into engram's validation/graph/health layer
- The real product path: extraction layer (done) → agent context injection (next)

## Alternatives Considered

1. **Write-time protocol** (like Contextual Commits) — rejected: requires behavior change, can't work retroactively
2. **Rule-based extraction** (no LLM) — rejected: commit messages are too terse for structured extraction without language understanding
3. **Full SaaS platform** — rejected: too heavy for MVP, Chromebook constraints, premature
