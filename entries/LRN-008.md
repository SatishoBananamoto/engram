# Chromebook constraints force better engineering decisions

## Context

All work happens on a Chromebook via Crostini (Linux container). Limited RAM, limited CPU, no GPU.

## What Happened

When choosing technologies for svx, persona-engine, and engram, the Chromebook constraint eliminated entire categories of tools: no numpy-heavy computation, no Docker, no large model inference locally. This forced:
- Pure Python with zero dependencies (engram)
- Lightweight agent patterns instead of heavy orchestration (svx)
- Text-based analysis instead of visualization libraries (all projects)

## Insight

Constraints aren't limitations — they're design decisions made by the environment. The Chromebook forces simplicity. Engram has zero dependencies because it had to, and that turned out to be a feature (portable, fast, no install steps). The predecessor had zero dependencies too but chose it as a "design principle." There's a difference between choosing simplicity and having it forced on you — the forced version is more honest.

## Applies To

Every technology decision. Ask "will this work on the Chromebook?" before adopting it. If not, find a simpler way.
