# Parser silently ignored entries with non-standard format fields

## Context

The calibration parser was designed to extract prediction entries from markdown format, but was reporting fewer verified predictions (31/41) than actually existed in the data.

## What Went Wrong

The regex parser was too strict and silently skipped entries that had additional fields between **Confidence:** and **Actual:** (like **Source type:** from session 3 entries). This caused systematic underreporting of verified predictions.

## Root Cause

The parser used a rigid regex pattern that expected exact field ordering without tolerance for optional intermediate fields that were added in later sessions.

## Prevention

Modify parsers to be tolerant of format variations, especially optional fields. Consider explicit validation with clear error messages rather than silent skipping of malformed entries.
