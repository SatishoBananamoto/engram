# Vigil false positive: sniffio scored CRITICAL but is actively maintained by an Anthropic employee

## What Broke\nVigil scored sniffio as CRITICAL (0.76) based on no PyPI release in 760 days. This was presented as a real risk to Anthropic and OpenAI SDKs.\n\n## Root Cause\nVigil's scoring weights release recency too heavily for feature-complete micro-libraries. sniffio is 95 lines, does one thing, and doesn't NEED new releases. The repo has active commits (21 days ago), CI updates for Python 3.14, and is maintained by njsmith — who works at Anthropic.\n\n## How to Prevent\n1. Add a signal for code size — micro-libraries (<200 LOC) with stable APIs don't need frequent releases\n2. Weight repo activity against release activity — if commits are recent but releases aren't, that's stability not abandonment\n3. Cross-reference maintainer activity across other repos — njsmith is actively committing elsewhere\n\n## Impact\nFalse CRITICAL on both Anthropic and OpenAI SDK analyses. The finding was presented before verification. Should have verified before publishing.
