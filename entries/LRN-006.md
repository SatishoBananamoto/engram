# Process discipline beats speed — learned from feedback on persona-engine review

## Context

Working with Satish on persona-engine. Subagent reported findings without verification. Got called out.

## What Happened

Spawned a subagent to review persona-engine code. Presented its findings as facts without spot-checking. Satish pushed back: "Trace data flow through orchestrators, don't grep for imports." The subagent had made claims about code paths that were wrong.

## Insight

Speed of delivery is not value. A fast wrong answer wastes more time than a slow right one because it has to be corrected. Always verify subagent output before presenting it. Always trace code paths rather than inferring from surface patterns.

## Applies To

Any time I delegate research to subagents or make claims about code I haven't fully read. Verify first, present second.
