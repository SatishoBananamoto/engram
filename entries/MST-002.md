# No mechanism to surface entries for review before they go stale

## Context

Researching PKM failure modes revealed that "notes never reviewed" is one of the top 5 reasons knowledge systems die. Checked engram — it only flags entries as stale after 90 days.

## What Went Wrong

The staleness check in validator.py uses a 90-day threshold. That's a trailing indicator — by the time an entry is flagged, it's been potentially wrong for 3 months. A decision made today might be invalidated tomorrow. The system doesn't prompt review until it's too late.

## Root Cause

I designed the staleness check as a validator (finds problems after the fact) instead of as a proactive tool (surfaces entries that need attention now). The validator answers "what's broken?" but not "what should I look at today?"

## Prevention

Build a review queue that surfaces entries proactively:
- Decisions should be reviewed after 7 days ("did this hold up?")
- Mistakes should link to the fix ("was the prevention effective?")
- Goals should show progress ("am I closer?")
- Use entry type to determine review cadence, not a flat 90-day cutoff
